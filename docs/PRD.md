# PRD - 猜數字遊戲系統

## 1. 專案概述
- **背景與動機**：提供一個簡單、有趣且容易上手的網頁小遊戲，讓使用者在短暫的休息時間內獲得娛樂，同時也是練習邏輯推理的好方式。
- **目標用戶**：所有人（不限年齡與背景的遊戲愛好者或需要短暫放鬆的使用者）。
- **核心價值主張**：以最直覺的介面與即時的回饋，提供無負擔的休閒娛樂體驗。

## 2. 功能需求
以下列出本系統的核心功能與使用者故事：

1. **隨機數字生成與遊戲初始化**
   - 作為 **玩家**，我希望 **系統能在每次開始新遊戲時隨機產生一個 1-100 的數字**，以便 **每次遊玩都有不同的挑戰**。
2. **使用者輸入與驗證**
   - 作為 **玩家**，我希望 **能有一個明確的輸入框讓我輸入猜測的數字，並阻擋無效的輸入（如英文字母或不在範圍內的數字）**，以便 **確保遊戲順暢進行而不發生錯誤**。
3. **猜測結果即時提示（太大 / 太小）**
   - 作為 **玩家**，我希望 **送出猜測後，系統能立即提示我的數字是「太大」還是「太小」**，以便 **我能根據提示調整下一次的猜測方向**。
4. **猜測紀錄與次數顯示**
   - 作為 **玩家**，我希望 **畫面上能顯示我目前的猜測次數以及過去猜過的數字清單**，以便 **作為我後續猜測的參考依據，避免重複猜相同的數字**。
5. **遊戲結束與重新開始**
   - 作為 **玩家**，我希望 **當我猜中數字時，系統會顯示成功訊息與總共猜測次數，並提供「重新開始」的按鈕**，以便 **我可以方便地直接展開下一局遊戲**。

## 3. 非功能需求
- **技術限制**：
  - 後端採用 Python Flask 框架。
  - 前端使用 Jinja2 模板引擎進行渲染。
  - 資料儲存使用 SQLite 資料庫（用於記錄排行榜或歷史紀錄）。
- **效能與安全考量**：
  - 系統應在 1 秒內快速回應使用者的猜測結果，確保良好的互動體驗。
  - 確保每個使用者的遊戲狀態獨立（透過 Session 進行狀態管理），避免不同玩家之間的資料互相干擾。
  - 防範常見的安全漏洞（例如 XSS，確保前端顯示的輸入能被安全地跳脫處理）。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 系統隨機生成 1-100 數字。
  - 玩家輸入數字介面與結果提示（太大、太小、猜中）。
  - 猜中後可重新開始新遊戲。
- **Should Have (應該有)**：
  - 顯示玩家當局的猜測次數。
  - 顯示當局過去猜測過的歷史數字清單。
  - 輸入值的基本前端與後端驗證（限制只能輸入 1-100 的整數）。
- **Nice to Have (可以有)**：
  - 歷史最佳成績排行榜（使用 SQLite 記錄最少猜測次數的玩家名稱與紀錄）。
  - 難度選擇（例如簡單 1-50、普通 1-100、困難 1-500）。
